byte

byte这个单词是Java八种基本数据类型之一字节的关键字，在计算机存储中以字节为单位，8位比特(bit)组成一个字节。

若byte为正数，则其最大值为

若byte为负数，则其最大值为

故byte的取值范围是-127~127。

但这个结论与我们在各种Java教程中看到的取值范围不吻合，byte的真正取值范围是

-2^82^8-1 即是 -128127
1
这个多出来的-128是怎么用字节表示的呢，用下面代码打印出-128的二进制表示。

byte b1 = -128;
System.out.println(Integer.toBinaryString(b1));
//结果
11111111 11111111 11111111 10000000
1

3
4
截取最后8位，所以-128的二进制表示为10000000,从表面上看就是-0。

实际上我们将二进制1111 1111表示的数字打印出来也不是-127,而是-1，可用下面代码进行验证

byte b1 = (byte) 0B11111111;
System.out.println(b1);
//结果
-1
1
2
3
4
出现这种情形是因为计算机用补码的方式存储数值，关于补码详细介绍可参考这篇文章。

简而言之就是，正数的补码是自身，而负数的补码则是其绝对值按位取反再加1。

要注意-128是不能用常规的方式转换成补码的，只是规定了10000000这个值用来表示-128。

equals和== 的区别

基本数据类型: == 比较数值
引用数据类型: == 比较内存地址
对象: == 与 equals 没有区别

equals 方法是 java.lang.Object 类的方法。

有两种用法说明：

（1）对于字符串变量来说，使用==和equals()方法比较字符串时，其比较方法不同。

== 比较两个变量本身的值，即两个对象在内存中的首地址。
equals() 比较字符串中所包含的内容是否相同。

final

  1. final 可以修饰变量和属性，一旦初始化无法修改。
  2. 如果修饰属性，jvm无法自动初始化，需要自己进行初始化
  3. 可以修饰方法和类，该方法无法被子类重写
  4. 不可以修饰构造方法
  5. 可以修饰方法的形参，一旦修饰，参数无法更改
  6. 不能和抽象方法或抽象类一起使用

抽象(abstarct)

抽象类：abstarct class 类名
由于类不完整，所以无法直接构造对象，可以通过子类间接构建对象
如果抽象类中含有抽象方法，子类继承抽象类需要重写父类的抽象方法，将方法补充完整

抽象方法：只有声明，没有实现的方法

tip:如果一个类里有抽象方法，这个类一定是抽象类
如果一个类是抽象类，这个类可以没有抽象方法

接口

  1. 基本语法：interface 接口名称{ 规则属性 ， 规则的行为 }
  2. 接口其实是抽象的
  3. 规则的属性必须是固定值，而且不能修改
  4. 属性的行为的访问权限必须是公共的
  5. 属性应该是静态的
  6. 行为应该是抽象的

同步和异步

同步

多个线程访问同步方法时，只能一个一个访问

阻塞

wait 等待（Object成员方法） && sleep 休眠 （Thread静态方法）

使用方式
wait：只能使用在同步代码中
sleep：可以在任意地方使用

阻塞时间
wait：超时时间（超过时间会引发异常,需要被唤醒）
sleep：休眠时间（不会发生错误）

同步处理
wait：如果执行wait方法，那么其他线程有机会执行当前的同步操作
sleep：如果执行sleep方法，其他线程没有机会执行当前的同步操作

类加载器

java中的类主要分为3种：

  1. Java核心类库的类：String,Object
  2. JVM软件平台的开发商
  3. 自己写的类

类加载器也有三种：

  1. BootClassLoder : 启动类加载器(加载类时，采用操作系统平台语言实现)
  2. PlatformClassLoder ：平台类加载器
  3. AppClassLoder：应用类加载器
