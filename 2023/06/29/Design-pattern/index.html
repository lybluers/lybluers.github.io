设计模式

单例模式

懒汉式

懒汉式是在首次使用时才创建实例的单例模式。具体来说，懒汉式在getInstance方法中延迟创建实例，只有在第一次调用getInstance时才会创建对象。

懒汉式的优点是节省了资源，在实际使用时才会创建对象，但缺点是在多线程环境下需要考虑线程安全问题。

饿汉式

饿汉式是在类加载时就创建实例的单例模式。具体来说，饿汉式在类加载时就创建了一个实例，并且在整个程序运行期间都可以直接使用这个实例。

饿汉式的优点是简单直接，没有线程安全问题，缺点是可能会造成资源浪费，因为在程序运行期间始终存在一个实例。

其他方式：

  1. 双重检测锁（Double-Checked Locking）：在懒汉式的基础上进行优化，使用双重检测来避免多线程环境下创建多个实例的问题。
  2. 静态内部类（Static Inner Class）：通过静态内部类的特性，实现延迟加载并保证线程安全。
  3. 枚举（Enum）：枚举实现的单例模式是线程安全的，并且可以防止反射和序列化破坏单例。
  4. 容器（Container）：将每个实例都放入一个容器中，通过唯一的标识符获取对应的实例。

工厂模式（Factory Pattern）

通过工厂类来创建对象，而不是直接通过构造函数创建。实现工厂模式的关键是定义一个工厂类，该类负责创建对象的实例。

步骤：

  1. 定义一个产品接口或抽象类，该接口或抽象类定义了产品的共同属性和方法。
  2. 创建具体产品类，实现产品接口或继承抽象类，并实现具体的业务逻辑。
  3. 创建工厂类，该类负责创建产品的实例。工厂类可以是普通类，也可以是抽象类或接口。
  4. 在工厂类中定义一个方法，该方法根据传入的参数决定创建哪个具体产品的实例。
  5. 在客户端代码中使用工厂类来创建产品的实例，而不是直接使用具体产品类。

观察者模式（Observer Pattern）

观察者模式（Observer Pattern）是一种行为设计模式，它定义了一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖它的对象都会收到通知并自动更新。

实现方式：

  1. 定义主题接口（Subject）：主题接口定义了主题对象的基本方法，包括添加观察者、移除观察者和通知观察者等。
  2. 定义观察者接口（Observer）：观察者接口定义了观察者对象的更新方法，当主题对象状态发生改变时，观察者对象会调用该方法进行更新。
  3. 创建具体主题类（ConcreteSubject）：具体主题类实现了主题接口，维护了观察者对象的列表，并在状态改变时通知观察者。
  4. 创建具体观察者类（ConcreteObserver）：具体观察者类实现了观察者接口，定义了自己的更新逻辑。

装饰器模式（Decorator Pattern）

装饰器模式（Decorator Pattern）是一种结构设计模式，它允许你动态地给一个对象增加额外的功能，装饰器模式比继承更加灵活。

实现步骤：

  1. 定义组件接口（Component）：组件接口定义了被装饰者和装饰者的共同方法。
  2. 创建具体组件类（ConcreteComponent）：具体组件类实现了组件接口，是被装饰者的具体实现。
  3. 创建装饰器类（Decorator）：装饰器类实现了组件接口，并持有一个被装饰者对象的引用，它可以在调用被装饰者的方法前后进行额外的操作。
  4. 创建具体装饰器类（ConcreteDecorator）：具体装饰器类继承自装饰器类，实现了装饰器类定义的额外操作。
