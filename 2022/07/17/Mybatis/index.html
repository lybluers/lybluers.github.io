简介

持久层框架，用于简化 JDBC 开发

持久层：负责将数据保存到数据库的那一层代码

JavaEE三层架构：表现层，业务层，持久层

入门

  1. 创建 maven 项目

  2. 导入依赖项坐标（mybatis,junit,mysql-connection,logback,junit 等）

  3. 创建 mybatis 核心配置文件 – 连接信息，解决硬编码问题（在resource中创建 mybatis-config.xml）

    ​ 基本模板

  4. 编写SQL映射文件 – 统一管理sql语句（在resource中编写 xxxMapper.xml）

    ​ 基本模板

  5. 编写 pojo 类（在 main/java/com/Lyblue/pojo 当中创建）

    创建数据库使用的基本类型参数，并添加 get 和 set 和 toString 方法

  6. 编写测试类（在 main/java/com/Lyblue/ 当中创建）

environment

typeAliases

在Mapper.xml当中, resultType 值可以直接写pojo类名称，不区分大小写

其中xml当中的命名空间为接口的路径（例如：com.Lyblue.mapper.UserMapper）

其中接口中存放调用 mapper 的方法

  1. 在测试类中使用mapper调用

若封装类的名称和数据库的名称不一致，可用如下方法

  1. 直接在语句中起别名(使别名和封装类名称一致)

  2. 在mapper.xml当中设置别名(resultMap)

    随后将查询语句的 resultType 改为 resultMap ,内容为 resultMap 的 id
